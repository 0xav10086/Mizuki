# 🚀 云端配置托管终极指南：打造跨平台无感同步的私人代理网络

在多设备（Windows 电脑、Arch Linux、Android 手机 + Box for KernelSU）环境下，手动维护 Clash 的 YAML 配置文件是一场灾难。每次添加规则或更换机场，都需要在各个设备间来回复制、重启内核，极度繁琐且容易出错。

这篇文档将带你彻底打破本地维护的桎梏，通过 VPS 搭建一个 **“全云端托管 + 自动化编译”**的代理控制中心。

## 📋 前置条件与环境准备（必读）

**推荐环境**：
- VPS：Ubuntu 22.04 / Debian 12 或 Arch Linux（推荐 1 核 1G 以上）
- 已安装：`nginx`、`curl`、`cron`（或 `crontab`）、`certbot`（已申请 Let’s Encrypt 证书）
- 拥有独立域名（例如 `yourdomain.com`），并已将 `dash.yourdomain.com` 解析到 VPS
- 本地已准备好一份完整的 Clash 配置文件（用于提取私人规则）

**替换提示**：
本文所有出现 `0xav10086.space` 的地方，请全部替换为你的域名。
脚本中的路径 `/opt/flowcollect/` 可根据需要修改，但需保持一致。

**安全建议**：
1. 运行脚本前执行 `chmod 700 /opt/flowcollect/*.sh`
2. Nginx 仪表盘路径建议添加 IP 白名单或 basic auth
3. 定期备份 `/opt/flowcollect/web/` 目录

## 🌟 架构优势：为什么这是终极优雅方案？

1. **一处修改，全端同步**：所有配置（节点、分流规则、策略组）全部存放在 VPS。本地设备只需填入一个固定的订阅链接，日后增删规则，只需在云端修改，所有设备自动拉取生效。
    
2. **极速加载（0 客户端运算）**：彻底摒弃在客户端让 Clash 去解析复杂的 `domain` 格式和庞大的外置规则库。VPS 会充当“持续集成服务器（CI）”，将所有上游规则清洗、格式化并编译为 7 个标准的 `classical` 模块。客户端只需“无脑”加载，启动和重载速度成倍提升。
    
3. **彻底解耦，绝对安全**：私人手写规则（如内网域名、特定网站）与云端开源规则集（如 Google、Steam、去广告）被巧妙的锚点隔离。VPS 脚本自动更新云端规则时，绝对不会覆盖你的私人规则。
    
4. **无痛应对“阅后即焚”**：VPS 脚本负责拿着一次性 Token 去机场提货并固化成节点文件，完美解决某些客户端无法直接更新一次性订阅链接的问题。

## 🛠️ 第一阶段：VPS 基础设施与 Nginx 配置

首先，在 VPS 上建立清晰的目录结构，并配置 Nginx 以确保 Clash 客户端能正确识别下载的文件。

### 1. 目录结构设计

在 VPS 上创建专属的工作目录：

```Bash
mkdir -p /opt/flowcollect/web/RuleSet
```

- `/opt/flowcollect/web/`：存放主配置文件、节点文件 (`nodes.yaml`) 和规则清单 (`86_rule_set_collect.csv`)。
    
- `/opt/flowcollect/web/RuleSet/`：专门存放编译后的 7 个终极规则集文件。

## 📦 第二阶段：自动化节点搬运工

### 获取机场的配置文件

编写一个脚本，负责将机场的订阅链接拉取并固化到本地。

新建 `/opt/flowcollect/get_nodes.sh`：

```Bash
#!/bin/bash

# 定义颜色输出，让终端提示更直观
GREEN='\033[0;32m'
YELLOW='\033[1;33m'
RED='\033[0;31m'
NC='\033[0m' # 恢复默认颜色

URL=""
NODE_NAME="nodes.yaml" # 默认名称

# 1. 智能参数解析：判断哪个参数是链接，哪个是名称
if [[ "$1" =~ ^https?:// ]]; then
        URL="$1"
        [ -n "$2" ] && NODE_NAME="$2"
elif [[ "$2" =~ ^https?:// ]]; then
        URL="$2"
        NODE_NAME="$1"
else
        # 如果两个都不是链接，仅保存第一个作为名字（随后抛出错误）
        [ -n "$1" ] && NODE_NAME="$1"
fi

# 2. 强制校验订阅链接
if [ -z "$URL" ]; then
        echo -e "${RED}❌ 错误：未检测到有效的订阅链接 (必须包含 http:// 或 https://)。${NC}"
        echo -e "用法 1: ./get_nodes.sh <订阅链接> [期望文件名]"
        echo -e "用法 2: ./get_nodes.sh [期望文件名] <订阅链接>"
        exit 1
fi

# 3. 规范化文件名 (如果没有 .yaml 后缀，自动补全)
if [[ ! "$NODE_NAME" == *.yaml ]]; then
        NODE_NAME="${NODE_NAME}.yaml"
fi

OUTPUT_FILE="/opt/flowcollect/web/$NODE_NAME"

echo -e "${YELLOW}⏳ 正在从机场获取最新节点...${NC}"
echo -e "🔗 链接: $URL"
echo -e "📦 目标: $OUTPUT_FILE"

# 4. 执行下载
curl -sS -A "clash.meta" "$URL" -o "$OUTPUT_FILE"

if [ $? -ne 0 ]; then
        echo -e "${RED}❌ 拉取失败，请检查网络或机场链接是否有效。${NC}"
        exit 1
fi

echo -e "${GREEN}✅ 节点文件已成功拉取！${NC}"

# 5. Nginx 自动化配置注入
NGINX_CONF="/etc/nginx/sites-available/my_sites"

# 检查 Nginx 文件是否存在
if [ ! -f "$NGINX_CONF" ]; then
        echo -e "${YELLOW}⚠️ 未找到 Nginx 配置文件 ($NGINX_CONF)，请手动将以下内容添加至仪表盘 server 块中：${{NC}"
        cat <<EOF
    location = /$NODE_NAME {
        alias $OUTPUT_FILE;
        default_type text/plain;
    }
EOF
        exit 0
fi

# 检查 Nginx 配置文件中是否已经存在该节点的路由
if grep -q "location = /$NODE_NAME" "$NGINX_CONF"; then
        echo -e "${GREEN}✅ 检测到 Nginx 已存在 /$NODE_NAME 的路由，无需重复添加。${NC}"
        echo -e "🎉 你的专属订阅链接为: ${YELLOW}https://dash.0xav10086.space/$NODE_NAME${NC}"
else
        echo -e "${YELLOW}⚙️ 正在自动修改 Nginx 配置...${NC}"

        # 使用 sed 在仪表盘的 server_name 下方安全注入 location 块
        sed -i '/server_name dash\.0xav10086\.space;/a \
    \
    # 自动生成的订阅节点下发配置 ('"$NODE_NAME"')\
    location = /'"$NODE_NAME"' {\
        alias '"$OUTPUT_FILE"';\
        default_type text/plain;\
    }' "$NGINX_CONF"

        # 测试 Nginx 语法
        nginx -t >/dev/null 2>&1
        if [ $? -eq 0 ]; then
                systemctl reload nginx
                echo -e "${GREEN}✅ Nginx 配置已成功注入并重载！${NC}"
                echo -e "🎉 你的专属订阅链接为: ${YELLOW}https://dash.0xav10086.space/$NODE_NAME${NC}"
        else
                echo -e "${RED}❌ Nginx 语法测试失败，请立刻使用 'nano $NGINX_CONF' 检查文件！${NC}"
        fi
fi
```

_赋予权限：`chmod +x /opt/flowcollect/get_nodes.sh`_

如果你的机场无法通过本脚本拉取下来，也可以将机场提供的订阅地址拉取到本地后，将机场的配置文件推送到与该脚本同级的路径中。

### 获取私人规则骨架

编写一个脚本，负责以自己的配置文件中的规则来创建规则骨架。这要求在运行脚本的同级目录下，存在你的配置文件，记得修改第三行的内容。

新建 `/opt/flowcollect/parse_private_rules.sh`：

```bash
#!/bin/bash

INPUT_FILE="fscn-master.yaml"

if [ ! -f "$INPUT_FILE" ]; then
        echo "❌ 找不到源文件 $INPUT_FILE"
        exit 1
fi

echo "⏳ 开始提取私人规则..."

# 初始化 7 个纯净的规则集文件
init_file() {
        local file=$1
        echo "payload:" >"$file"
        echo "  # ==========================================" >>"$file"
        echo "  # [MANUAL_START] Private" >>"$file"
        echo "  # ==========================================" >>"$file"
}

init_file "86JPRules.yaml"
init_file "86HKRules.yaml"
init_file "86USRules.yaml"
init_file "86BemlyRules.yaml"
init_file "86SwitchRules.yaml"
init_file "86DirectRules.yaml"
init_file "86RejectRules.yaml"

get_target_file() {
        case "$1" in
        Japan) echo "86JPRules.yaml" ;;
        HK) echo "86HKRules.yaml" ;;
        US) echo "86USRules.yaml" ;;
        bemly) echo "86BemlyRules.yaml" ;;
        Switch) echo "86SwitchRules.yaml" ;;
        DIRECT) echo "86DirectRules.yaml" ;;
        REJECT) echo "86RejectRules.yaml" ;;
        *) echo "" ;;
        esac
}

in_rules=0
while IFS= read -r line; do
        line=$(echo "$line" | tr -d '\r')
        if [[ "$line" =~ ^rules: ]]; then
                in_rules=1
                continue
        fi
        if [[ $in_rules -eq 1 && "$line" =~ ^[a-zA-Z] ]]; then in_rules=0; fi

        if [[ $in_rules -eq 1 && "$line" =~ ^[[:space:]]*-[[:space:]]*(.+) ]]; then
                rule_content="${BASH_REMATCH[1]}"
                rule_content="${rule_content//\"/}"
                rule_content="${rule_content//\'/}"
                rule_content="${rule_content%%#*}"
                rule_content=$(echo "$rule_content" | xargs)

                if [[ -z "$rule_content" ]]; then continue; fi

                clean_rule="${rule_content%,no-resolve}"
                target="${clean_rule##*,}"
                payload="${clean_rule%,*}"
                type="${payload%%,*}"

                if [[ "$rule_content" == *",no-resolve" ]]; then
                        payload="${payload},no-resolve"
                fi

                target_file=$(get_target_file "$target")
                if [[ -z "$target_file" ]]; then continue; fi

                # 只提取非系统内置且非 RULE-SET 的私人规则
                if [[ "$type" != "RULE-SET" && "$type" != "GEOIP" && "$type" != "MATCH" && "$type" != "NETWORK" ]]; then
                        echo "  - $payload" >>"$target_file"
                fi
        fi
done <"$INPUT_FILE"

# 封口
close_file() {
        local file=$1
        echo "  # ==========================================" >>"$file"
        echo "  # [MANUAL_END] Private" >>"$file"
        echo "  # ==========================================" >>"$file"
        echo "✅ 已生成纯私人规则: $file"
}

for f in 86*.yaml; do
        if [[ "$f" != "$INPUT_FILE" ]]; then close_file "$f"; fi
done

echo "🎉 私人规则提取完成！请将这 7 个文件上传到 VPS 的 /opt/flowcollect/web/RuleSet 目录。"
```

### 获取私人规则集清单

编写一个脚本，负责以自己的配置文件中的规则来创建规则集配置表单。这要求在运行脚本的同级目录下，存在你的配置文件，记得修改第三行的内容。

新建 `/opt/flowcollect/generate_csv.sh`：

```
#!/bin/bash

INPUT_FILE="fscn-master-2604031459.yaml"
OUTPUT_FILE="86_rule_set_collect.csv"

if [ ! -f "$INPUT_FILE" ]; then
        echo "❌ 找不到源文件 $INPUT_FILE"
        exit 1
fi

echo "⏳ 正在解析 $INPUT_FILE，生成云端规则收集表..."

# 声明关联数组用于存储 provider 的详细信息
declare -A urls
declare -A behaviors
declare -A paths
declare -A targets

in_providers=0
current_provider=""

# ==========================================
# 第一阶段：提取 rule-providers 信息
# ==========================================
while IFS= read -r line; do
        line=$(echo "$line" | tr -d '\r') # 剔除 Windows 回车符
        if [[ "$line" =~ ^rule-providers: ]]; then
                in_providers=1
                continue
        fi
        if [[ $in_providers -eq 1 && "$line" =~ ^[a-zA-Z] ]]; then in_providers=0; fi

        if [[ $in_providers -eq 1 ]]; then
                # 匹配 provider 名称
                if [[ "$line" =~ ^[[:space:]]{2}([A-Za-z0-9_-]+): ]]; then
                        current_provider="${BASH_REMATCH[1]}"
                # 匹配 behavior
                elif [[ "$line" =~ behavior:[[:space:]]*([a-zA-Z]+) ]]; then
                        behaviors["$current_provider"]="${BASH_REMATCH[1]}"
                # 匹配 url (智能去引号)
                elif [[ "$line" =~ url:[[:space:]]*[\"\']?([^\"\']+)[\"\']? ]]; then
                        urls["$current_provider"]="${BASH_REMATCH[1]}"
                # 匹配 path (智能去引号)
                elif [[ "$line" =~ path:[[:space:]]*[\"\']?([^\"\']+)[\"\']? ]]; then
                        paths["$current_provider"]="${BASH_REMATCH[1]}"
                fi
        fi
done <"$INPUT_FILE"

# ==========================================
# 第二阶段：提取 rules 中规则集的映射关系
# ==========================================
in_rules=0
while IFS= read -r line; do
        line=$(echo "$line" | tr -d '\r')
        if [[ "$line" =~ ^rules: ]]; then
                in_rules=1
                continue
        fi
        if [[ $in_rules -eq 1 && "$line" =~ ^[a-zA-Z] ]]; then in_rules=0; fi

        if [[ $in_rules -eq 1 ]]; then
                # 清除可能存在的双引号或单引号 (比如 - "RULE-SET,Github,bemly")
                clean_line="${line//\"/}"
                clean_line="${clean_line//\'/}"

                # 匹配 RULE-SET 映射 (格式: - RULE-SET,ProviderName,TargetGroup)
                if [[ "$clean_line" =~ ^[[:space:]]*-[[:space:]]*RULE-SET,([^,]+),([^,]+) ]]; then
                        provider_name="${BASH_REMATCH[1]}"
                        target="${BASH_REMATCH[2]}"

                        # 清理可能带有的 no-resolve 或注释
                        target="${target%%,*}"
                        target="${target%%#*}"

                        # 去除首尾空格
                        target=$(echo "$target" | xargs)
                        provider_name=$(echo "$provider_name" | xargs)

                        targets["$provider_name"]="$target"
                fi
        fi
done <"$INPUT_FILE"

# ==========================================
# 第三阶段：组装并生成 CSV 文件
# ==========================================
# 第一行写入带 # 的表头，方便人类阅读，同时也让 VPS 脚本安全跳过
echo "#Name,TargetGroup,Behavior,LocalPath,URL" >"$OUTPUT_FILE"

for name in "${!urls[@]}"; do
        target="${targets[$name]}"

        # 防止误提取你已经写好的 86 系列
        if [[ "$name" == 86* ]]; then continue; fi

        if [[ -n "$target" ]]; then
                echo "$name,$target,${behaviors[$name]},${paths[$name]},${urls[$name]}" >>"$OUTPUT_FILE"
                echo "  -> 已记录: $name => 将路由至 $target"
        else
                echo "  ⚠️ 警告: $name 未在 rules 路由表中使用，已跳过。"
        fi
done

echo "----------------------------------------"
echo "🎉 收集表生成完毕！请检查 $OUTPUT_FILE 文件。"
```

## 🧠 第三阶段：全云端规则编译中心（核心！）

这是整个架构的灵魂。我们将所有流量归类为 7 个终点（如 `Japan`, `HK`, `DIRECT`, `REJECT` 等），并为它们分别创建规则骨架。

### 1. 私人规则骨架 (`86XXRules.yaml`)

在 `/opt/flowcollect/web/RuleSet/` 下创建 7 个文件（例如 `86JPRules.yaml`）。它们必须包含极其严格的 `[MANUAL_START]` 和 `[MANUAL_END]` 锚点。在上一节中，我们已经生成了这部分内容，你需要做的就是将其放置在对应目录。

**`86JPRules.yaml` 示例：**

```YAML
payload:
  # ==========================================
  # [MANUAL_START] Private
  # ==========================================
  - DOMAIN-KEYWORD,grok
  - DOMAIN-KEYWORD,nhentai
  # ==========================================
  # [MANUAL_END] Private
  # ==========================================
```

### 2. 开源规则清单 (`86_rule_set_collect.csv`)

在 `/opt/flowcollect/web/` 下创建 `86_rule_set_collect.csv`，作为你的“规则数据库”。格式为：`名称,目标策略组,原始格式,占位,云端链接`。在上一节中，我们已经生成了这部分内容，你需要做的就是将其放置在对应目录。

```code
#Name,TargetGroup,Behavior,LocalPath,URL
Apple,DIRECT,classical,,https://cdn.jsdelivr.net/gh/blackmatrix7/ios_rule_script@master/rule/Clash/Apple/Apple.yaml
Google,Japan,domain,,https://cdn.jsdelivr.net/gh/Loyalsoldier/clash-rules@release/google.txt
Steam,Switch,classical,,https://raw.githubusercontent.com/blackmatrix7/ios_rule_script/master/rule/Clash/Steam/Steam.yaml
```

### 3. 极速编译脚本 (`update_rules.sh`)

摒弃效率低下的 Bash 循环，采用底层 AWK/SED 流处理，**1秒内即可清洗并重组几十万条规则**，自动将 `domain` 格式转换为统一的 `classical` 格式。

新建 `/opt/flowcollect/web/update_rules.sh`：

```bash
#!/bin/bash

WEB_DIR="/opt/flowcollect/web"
RULE_DIR="$WEB_DIR/RuleSet"
CSV_FILE="$WEB_DIR/86_rule_set_collect.csv"

if [ ! -f "$CSV_FILE" ]; then
    echo "[Error] 找不到规则收集表: $CSV_FILE"
    exit 1
fi

echo "⏳ 开始编译并更新全量规则集..."

# 1. 初始化：清理所有规则集 [MANUAL_END] 之后的内容，防止重复追加
for file in "$RULE_DIR"/86*.yaml; do
    if [ -f "$file" ]; then
        # sed 命令：找到 [MANUAL_END] Private 这一行，并在这行退出匹配，删除这行之后的所有内容
        sed -i '/\[MANUAL_END\] Private/q' "$file"
        echo "" >> "$file" # 补充一个空行
    fi
done

# 辅助函数：根据目标策略组返回对应的文件名
get_target_file() {
    case "$1" in
        Japan)   echo "86JPRules.yaml" ;;
        HK)      echo "86HKRules.yaml" ;;
        US)      echo "86USRules.yaml" ;;
        bemly)   echo "86BemlyRules.yaml" ;;
        Switch)  echo "86SwitchRules.yaml" ;;
        DIRECT)  echo "86DirectRules.yaml" ;;
        REJECT)  echo "86RejectRules.yaml" ;;
        *)       echo "" ;;
    esac
}

# 2. 读取 CSV，下载并处理云端规则
# 使用逗号作为分隔符
while IFS=, read -r name target behavior local_path url || [ -n "$name" ]; do
    # 忽略空行和注释行
    name=$(echo "$name" | xargs)
    if [[ -z "$name" || "$name" == \#* ]]; then continue; fi

    target=$(echo "$target" | xargs)
    behavior=$(echo "$behavior" | xargs)
    url=$(echo "$url" | tr -d '\r' | xargs) # 去除可能带来的 Windows 回车符

    target_file="$RULE_DIR/$(get_target_file "$target")"

    if [ ! -f "$target_file" ]; then
        echo "[Warn] 找不到策略组 $target 对应的文件，跳过 $name"
        continue
    fi

    echo "-> 正在拉取并编译: $name (-> $target)"

    # 写入模块头部注释
    echo "  # === [AUTO: $name] ===" >> "$target_file"

    # 下载到临时文件
    curl -sS -L "$url" -o "$WEB_DIR/raw.tmp"

    # 根据行为(behavior)使用 AWK 极速清洗并追加到目标文件
    if [[ "$behavior" == "domain" ]]; then
        awk '
        /^#/ || /^payload:/ || /^[[:space:]]*$/ {next}
        {
           sub(/^[[:space:]]*-[[:space:]]*/, "");
           gsub(/['\''\"]/, "");
           if ($0 ~ /^\+\./) { print "  - DOMAIN-SUFFIX," substr($0, 3) }
           else if ($0 ~ /^\+/) { print "  - DOMAIN-SUFFIX," substr($0, 2) }
           else if ($0 ~ /^full:/) { print "  - DOMAIN," substr($0, 6) }
           else if ($0 ~ /^domain:/) { print "  - DOMAIN-SUFFIX," substr($0, 8) }
           else { print "  - DOMAIN-SUFFIX," $0 }
        }' "$WEB_DIR/raw.tmp" >> "$target_file"
    else
        awk '
        BEGIN { in_payload=0 }
        /^payload:/ { in_payload=1; next }
        in_payload && /^[[:space:]]*-[[:space:]]*/ { print $0 }
        ' "$WEB_DIR/raw.tmp" >> "$target_file"
    fi

    echo "  # =====================" >> "$target_file"
    echo "" >> "$target_file"

done < "$CSV_FILE"

# 清理临时文件
rm -f "$WEB_DIR/raw.tmp"
echo "[Success] 所有规则集编译完成！你的客户端现在可以拉取到满血版本了。"
```

### 4. 彻底的自动化（Cron Job）

让 VPS 成为全自动管家。创建以下脚本。

```bash
#!/bin/bash

# ==========================================
# Global Configuration
# ==========================================
WEB_DIR="/opt/flowcollect/web"
RULE_DIR="$WEB_DIR/RuleSet"
CSV_FILE="$WEB_DIR/86_rule_set_collect.csv"
LOG_FILE="/opt/flowcollect/auto_update.log"

# Redirect all stdout and stderr to the log file and also print to terminal
exec > >(tee -a "$LOG_FILE") 2>&1

echo "==================================================="
echo "🚀 [$(date '+%Y-%m-%d %H:%M:%S')] Auto-Update Task Started"
echo "==================================================="

# ==========================================
# Phase 1: Update Regular Airport Subscriptions
# ==========================================
# Define an associative array (dictionary) for standard airports.
# Format: ["filename"]="subscription_url"
declare -A AIRPORT_URLS
AIRPORT_URLS=(
    ["bemly_node.yaml"]=""
    ["cf_node.yaml"]=""
)

echo "--> Phase 1: Updating standard subscriptions..."

# Iterate over the dictionary keys (filenames)
for file_name in "${!AIRPORT_URLS[@]}"; do
    url="${AIRPORT_URLS[$file_name]}"
    target_file="$WEB_DIR/$file_name"
    
    echo "⏳ Fetching nodes for [$file_name]..."
    
    # Download to a temporary file first, capture HTTP status code
    HTTP_CODE=$(curl -sS -L -w "%{http_code}" -A "clash.meta" "$url" -o "${target_file}.tmp")
    
    # Check if download was successful (HTTP 200)
    if [ "$HTTP_CODE" = "200" ]; then
        mv "${target_file}.tmp" "$target_file"
        echo "✅ Successfully updated [$file_name]."
    else
        # Remove the corrupted temp file if download fails
        rm -f "${target_file}.tmp"
        echo "❌ Failed to update [$file_name] (HTTP: $HTTP_CODE). Retaining old file."
    fi
done

# ==========================================
# Phase 2: Handle "Burn-After-Reading" Airport
# ==========================================
echo -e "\n--> Phase 2: Handling burn-after-reading subscriptions..."

# 'shanhuyun' relies on a burn-after-reading mechanism.
# It is intentionally excluded from the dictionary above to prevent 403 errors.
echo "⚠️  Skipping 'shanhuyun' (Burn-After-Reading mechanism)."
echo "   Please update it manually using the dedicated one-time link script when needed."

# ==========================================
# Phase 3: Compile and Update Rule Sets
# ==========================================
echo -e "\n--> Phase 3: Compiling cloud rule sets..."

if [ ! -f "$CSV_FILE" ]; then
    echo "❌ [Error] Rule collection CSV not found: $CSV_FILE"
    exit 1
fi

# Initialize: Clear content after [MANUAL_END] Private to prevent duplicate appends
for file in "$RULE_DIR"/86*.yaml; do
    if [ -f "$file" ]; then
        sed -i '/\[MANUAL_END\] Private/q' "$file"
        echo "" >> "$file" # Add an empty line for spacing
    fi
done

# Helper function: Map target policy group to filename
get_target_file() {
    case "$1" in
        Japan)   echo "86JPRules.yaml" ;;
        HK)      echo "86HKRules.yaml" ;;
        US)      echo "86USRules.yaml" ;;
        bemly)   echo "86BemlyRules.yaml" ;;
        Switch)  echo "86SwitchRules.yaml" ;;
        DIRECT)  echo "86DirectRules.yaml" ;;
        REJECT)  echo "86RejectRules.yaml" ;;
        *)       echo "" ;;
    esac
}

# Read CSV, download and process cloud rules
while IFS=, read -r name target behavior local_path url || [ -n "$name" ]; do
    name=$(echo "$name" | xargs)
    # Skip empty lines and commented lines
    if [[ -z "$name" || "$name" == \#* ]]; then continue; fi

    target=$(echo "$target" | xargs)
    behavior=$(echo "$behavior" | xargs)
    url=$(echo "$url" | tr -d '\r' | xargs)

    target_file="$RULE_DIR/$(get_target_file "$target")"
    
    if [ ! -f "$target_file" ]; then
        echo "⚠️  [Warn] Target file for policy group $target not found, skipping $name"
        continue
    fi

    echo "  -> Fetching and compiling: $name (-> $target)"
    
    # Write module header comment
    echo "  # === [AUTO: $name] ===" >> "$target_file"

    # Download to temp file
    curl -sS -L "$url" -o "$WEB_DIR/raw.tmp"
    
    # Process and append to target file using AWK based on behavior mapping
    if [[ "$behavior" == "domain" ]]; then
        awk '
        /^#/ || /^payload:/ || /^[[:space:]]*$/ {next}
        {
           sub(/^[[:space:]]*-[[:space:]]*/, "");
           gsub(/[\047\042]/, "");
           if ($0 ~ /^\+\./) { print "  - DOMAIN-SUFFIX," substr($0, 3) }
           else if ($0 ~ /^\+/) { print "  - DOMAIN-SUFFIX," substr($0, 2) }
           else if ($0 ~ /^full:/) { print "  - DOMAIN," substr($0, 6) }
           else if ($0 ~ /^domain:/) { print "  - DOMAIN-SUFFIX," substr($0, 8) }
           else { print "  - DOMAIN-SUFFIX," $0 }
        }' "$WEB_DIR/raw.tmp" >> "$target_file"
    else
        awk '
        BEGIN { in_payload=0 }
        /^payload:/ { in_payload=1; next }
        in_payload && /^[[:space:]]*-[[:space:]]*/ { print $0 }
        ' "$WEB_DIR/raw.tmp" >> "$target_file"
    fi
    
    echo "  # =====================" >> "$target_file"
    echo "" >> "$target_file"

done < "$CSV_FILE"

# Cleanup temp file
rm -f "$WEB_DIR/raw.tmp"
echo "✅ [Success] All rule sets successfully compiled!"

echo "==================================================="
echo "🎉 [$(date '+%Y-%m-%d %H:%M:%S')] Auto-Update Task Finished"
echo "==================================================="
echo ""
```

以下是设置定时任务的完整步骤：

### 1. 打开定时任务编辑器

在终端中输入以下命令来编辑当前用户的定时任务列表：

```bash
crontab -e
```

_(注：如果你平时习惯用 Neovim，可以通过预设环境变量来强制 `crontab` 使用它打开，例如 `EDITOR=nvim crontab -e`)_

### 2. 添加定时任务规则

进入编辑界面后，在文件的最末尾添加以下这行代码。这里我为你设定的是 **每天凌晨 4:30** 执行一次（这个时间点通常网络比较闲置，且能避开大多数服务的整点备份高峰）：

```Plaintext
30 4 * * * /opt/flowcollect/web/auto_update_node_and_rule.sh
```

**Cron 表达式解析：**

- `30`：分钟 (第 30 分钟)
    
- `4`：小时 (凌晨 4 点)
    
- `*`：日期 (每天)
    
- `*`：月份 (每个月)
    
- `*`：星期 (星期一到星期日)
    
- 最后一段是你的脚本绝对路径。由于脚本内部已经写好了日志重定向（`exec > >(tee -a "$LOG_FILE") 2>&1`），所以这里不需要再写多余的 `>>` 日志输出代码，非常整洁。
    

### 3. 保存并退出

- 如果你的默认编辑器是 **Vim/Neovim**：按 `Esc`，输入 `:wq` 并回车。
    
- 如果你的默认编辑器是 **Nano**：按 `Ctrl + O` 保存，回车确认，然后按 `Ctrl + X` 退出。
    

系统会提示 `crontab: installing new crontab`，这就说明任务添加成功了。

### 4. 验证任务是否添加成功

你可以通过以下命令查看当前已配置的定时任务列表：

```Bash
crontab -l
```
